# Revisión de ToS / robots.txt — Hepy

| Sitio | robots.txt | Términos de Servicio | Rate limit aplicado | Decisión |
|---|---|---|---|---|
| Superseis (superseis.com.py) | Bloqueado por CloudFront incluso para /robots.txt (403) | Confirmado en producción: bloqueado tanto por `requests` plano como por un navegador real (headless Chromium vía Playwright) — 403 antes de que cargue cualquier contenido. Es un bloqueo de red/fingerprint, no un problema de estrategia de scraping. | N/A | **No viable para v1.** Excluido de la canasta agregada (afecta solo cobertura, no invalida el resto); `run_daily.py` lo aísla sin abortar el resto de la corrida. Requeriría una red residencial no-datacenter o un setup de navegador stealth para reconsiderar. |
| Stock (stock.com.py) | HTTP 200 | Confirmado en producción: NO es nopCommerce (solo conserva un comentario de copyright viejo) — es un storefront ASP.NET WebForms a medida. El path real de búsqueda es `search.aspx?searchterms=<term>` (encontrado vía un link de promo en vivo), no `/search?q=` como se había asumido. Sin JSON-LD; se parsea el HTML real (`div.product-item` / `span.productPrice` / `h2.product-title`). | 1 request cada 2-3s, User-Agent identificado con contacto | Permitido, implementado y verificado con datos reales |
| Casa Rica (casarica.com.py) | Sin archivo robots.txt (404) | Confirmado en producción: comparte el mismo tema WooCommerce "EcommercePro" que Areté y Los Jardines. El formulario de búsqueda rápida (`?q=` sobre `catalogo`) NO filtra resultados (verificado: términos distintos devuelven la misma salida); el formulario real postea a `productos?q=<term>&post_type=product` y sí filtra. Sin JSON-LD. | 1 request cada 2-3s | Permitido, implementado y verificado con datos reales |
| Salemma (salemmaonline.com.py) | HTTP 200, disallow de carrito/checkout/cuenta/login; permite `?page=` | Confirmado en producción: sin JSON-LD en ninguna página (solo schema `WebSite`/`LocalBusiness`); el path real de búsqueda (`/buscar?q=`, no `/busqueda?q=`) se recuperó del propio `SearchAction` de ese schema `WebSite`. Se parsean las tarjetas `div.divproduct` reales. | 1 request cada 2-3s | Permitido, implementado y verificado con datos reales |
| Biggie (biggie.com.py) | Redirect a biggie.com.py | Confirmado en producción: es una SPA (Vue), sin HTML server-rendered ni JSON-LD. El frontend llama directamente a una API JSON propia (`api.app.biggie.com.py/api/articles`), encontrada por inspección de red con navegador real. Se usa esa API directamente. | 1 request cada 2-3s | Permitido, implementado y verificado con datos reales |
| Areté (arete.com.py) | Sin archivo robots.txt (404) | Confirmado en producción: mismo tema WooCommerce "EcommercePro" que Casa Rica y Los Jardines; búsqueda real en `productos?q=<term>&post_type=product`. Sin JSON-LD. | 1 request cada 2-3s | Permitido, implementado y verificado con datos reales |
| Grütter (grutteronline.casagrutter.com.py) | Inalcanzable desde el sandbox de planificación (fallo de conexión, no 403/404) | Confirmado en producción: el sitio SÍ es alcanzable (la falla de conexión no se reprodujo desde este entorno). WooCommerce estándar (tema "megashop"), búsqueda vía `?s=<term>&post_type=product` filtra correctamente. Sin JSON-LD. | 1 request cada 2-3s | Permitido, implementado y verificado con datos reales |
| Los Jardines (losjardinesonline.com.py) | Sin archivo robots.txt (404) | Confirmado en producción: mismo tema WooCommerce "EcommercePro"; a diferencia de Casa Rica, su formulario real SÍ postea correctamente a `catalogo?q=<term>`. Sin JSON-LD. | 1 request cada 2-3s | Permitido, implementado y verificado con datos reales |
| Real (realonline.com.py) | HTTP 200, declara `Content-Signal: search=yes,ai-train=no,use=reference` | Confirmado en producción: sitio Next.js sin API de búsqueda invocable desde el navegador (el backend GraphQL solo expone autocompletado/tracking). Los datos reales vienen del payload RSC (React Server Components) que Next.js entrega ante un header `RSC: 1` — confirmado que funciona desde `requests` plano, sin necesidad de navegador. | 1 request cada 2-3s | **Permitido explícitamente para "reference"** — citar este directive en el paper/README como evidencia de buena fe; implementado y verificado con datos reales |

Regla general aplicada a los 9 scrapers: identificarse con un User-Agent propio
(`Hepy/0.1 (+https://github.com/<org>/hepy; contacto: <email>)`), 1 request cada
2-3 segundos por sitio, nunca en paralelo entre sitios distintos en la misma
ventana de tiempo, y respetar cualquier `Disallow` de robots.txt aunque el
scraping general esté permitido.
